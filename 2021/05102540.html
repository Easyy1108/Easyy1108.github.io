Android 复习总结

Android
发布日期:   2021-05-10

安卓知识点复习总结

一、Android体系结构

  1. 应用程序层 Application
    平常所用的Android应用程序都是在这一层上，这层开发的应用程序都是Java语言编写的，调用application framework提供的api完成。

  2. 应用程序框架层 application framework
    这层为开发人员提供了完全访问核心应用程序所使用的api框架

  3. 核心类库 libraries
    核心类库包含了系统提供一些核心库，允许开发者使用java语言编写Android application还包括了dalvik 虚拟机。

  4. Linux内核
    内核作为一个抽象层，位于软件栈层和硬件层之间。



二、模拟器创建的过程

  1. 打开Android studio ，点击图标创建模拟器按钮
  2. create virtual device
  3. 点击左侧类别，创建自己的模拟器
  4. 选择操作系统
  5. 运行项目

三、 Android 程序结构

  • manifests：存放AndroidManifest.xml
  • Java ：存放所有的Java代码
  • res：存放Android程序所用到的资源，如图片、布局文件、字符串
  • GrandleScripts：用于存放项目创建的相关文件

四、什么是布局？

布局全名为 布局管理器，是对ViewGroup类的扩展，用来控制子空间在ui中的位置。



五、布局的类型有哪些？特点？控件属性？

线性布局

线性布局以水平和垂直方式来显示界面中的ui控件
水平排列：显示顺序依次为从左到右
垂直排列：显示顺序依次为从上到下

重要属性：orientation

相对布局 relativelayout

  Android应用程序创建时默认的额布局，相对布局是通过相对定位的额方式指定控件位置。

控件属性 功能描述
android:layout_centerInParent 设置当前控件位于父布局的中央位置
android:layout_centerVetical 设置当前控件位于父布局的垂直居中
android:layout_centerHorizontal 设置当前控件位于父布局的额水平居中

android:layout_above 设置当前控件位于某控件上方
android:layout_below 设置当前控件位于某控件下方
android:layout_toLeftOf 设置当前控件位于某控件左侧
android:layout_toRightOf 设置当前控件位于某控件右侧

android:layout_alignParentTop 设置当前控件是否位与父控件顶端对齐
android:layout_alignParentLeft 设置当前控件是否位与父控件左对齐
android:layout_alignParentRight 设置当前控件是否位与父控件右对齐
android:layout_alignParenBottom 设置当前控件是否位与父控件底端对齐

android:layout_alignTop 设置当前控件的上边界是否与某控件对齐
android:layout_alignBottom 设置当前控件的下边界是否与某控件对齐
android:layout_alignLeft 设置当前控件的左边界是否与某控件对齐
android:layout_alignRight 设置当前控件的右边界是否与某控件对齐

帧布局 frameLayout

  帧布局是Android中最为常见的一种布局，该布局为每个加入其中的控件创建一个空白区域，采用帧布局方式设计界面时，所有控件都默认显示在左上角，并按照先后放入的顺序重叠摆放。

控件属性 功能描述
andriod:foreground 设置帧布局容器的前景图片
android:foregroundGravity 设置前景图片显示位置 
andriod:foreground="@mipmap/ic_launcher"
andriod:foregroundGravity="left"

表格布局 tablelayout

表格布局是以表格形式排列控件的，通过行列将界面划分为多个单元格，每个单元格都可以添加控件。表格布局需要配合TableRow对象组成，因此TableRow的数量决定表格的行数。


布局属性 功能描述
android:stretchColumns 设置该列被拉伸，列号从“0”开始。
android:shrinkColumns 设置该列被收缩，列号从“0”开始
android:collapseColumns 设置该列被隐藏，列号从“0”开始

控件属性 功能描述
android:layout_column 设置改单元显示位置
android:layout_span 设置该单元格占几行，默认一行

Tips layout_width 、layout_height 、 width 、height的区别

在Android系统中，layout width、 layout height 属性和width、height 属性的功能是相同的，都用于设置控件的宽、高，只不过带”layout” 前缀的属性通常是相对父控件而言的，而width.height属性是相对于控件本身而言的。下面对比一下它们在使用时的区别。

  1. layout, width 和layout heigt 属性可以单独使用，而widh和heigt属性不能，如果单独使用width和height属性，此时的控件是不显示的。

  2. layout width和layout height可以设置为wrap content或者match_ parent, 而width和height只能设置固定值，否则会产生编译错误。

  3. 如果要使用width和height, 就必须同时设置layout _width和layout height 属性，把width和height作为组件的微调使用。

简单方便。综上所述，在设置TexView控件宽、高时，通常直接使用layout_width和layout _height.
